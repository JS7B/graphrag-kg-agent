# AGENTS.md 

# 个人知识图谱 GraphRAG Agent

面向个人文档集合（技术论文 / GitHub 仓库文档 / 产品需求文档）的端到端 GraphRAG 系统：自动完成文档解析、实体与关系抽取、Neo4j 知识图谱构建、向量召回 + 图谱邻域扩展检索，生成**可追溯引用**的回答，并配套一个清晰专业、带像素 Agent 动效的前端工作台。定位为个人学习 + 简历展示 + 公开可复现项目。

**技术栈**：Python 3.11+ / FastAPI / Pydantic（后端 API 与 LLM 结构化输出）· Neo4j + Vector Index（图谱与向量存储，Docker 本地部署）· OpenAI-compatible chat & embedding（LLM）· React + Vite + TypeScript（前端）· Docker Compose（编排）。

> **本文件面向通用编码 Agent（如 codex）。** Claude Code 请读同步副本 `CLAUDE.md`；通用编码规范、个人环境约定见全局 `~/.claude/CLAUDE.md`。此处只覆盖本方案的导航、硬规则、决策边界与学习记录约定。

## 硬性规则

- **密钥与数据零提交**：密钥走 `.env`（仓库只提供 `.env.example`）；`.gitignore` 必须覆盖密钥、缓存、运行数据、Neo4j 数据目录、本地样本文档。公开仓库，从第一天起配置。
- **模型配置全部走环境变量**：`OPENAI_BASE_URL` / `OPENAI_API_KEY` / `CHAT_MODEL` / `EMBEDDING_MODEL`，严禁硬编码模型名或地址。
- **LLM 统一用 OpenAI-compatible 接口**；图谱存储统一用 Neo4j，向量召回用 Neo4j Vector Index，不另起一套向量库。
- **引用可追溯是硬要求**：每个 entity / relation 必须能回到来源 chunk 或文档位置；回答必须带引用，引用必须能落到原始 chunk。
- **像素 Agent 动画状态必须来自真实 `RunEvent`**，不得用前端伪造状态驱动。
- **目录保持 git worktree 友好**：运行数据、缓存、Neo4j 数据目录不得误入版本控制。

## 决策边界

- **偏离 ≠ 冻结**：概念模型、Neo4j schema、API 命名与响应结构、实体/关系类型在规划阶段**不定死**，围绕样本文档和评估集逐步收敛；不为"图谱看起来复杂"而堆无用类型。业务关系先统一落 `:RELATES`，关系类型作属性，稳定后再判断是否拆分。
- **GraphRAG 接入但保留项目级控制权**：检索策略、上下文组装、证据追踪、回答结构、前端事件输出统一由本项目编排层管理，不被第三方框架的默认行为牵着走。
- **PDF 范围**：文本型 PDF 必须落地；扫描版 OCR **不作为核心验收**，但解析层保留扩展口，避免主线跑偏成 OCR 工程。
- **简单优先**：任务执行先用简单后台任务/任务表，确有必要再引入 Celery / RQ / Dramatiq；前端样式、图谱可视化库、像素动画都以简单稳定为先，避免过早引入重型方案。
- **先跑通再命名**：API 按"先打通核心链路，再补齐命名与响应结构"推进，不在早期冻结接口。

## 学习记录约定（DEVLOG）

项目所有者是在学习中开发，对 Docker、Neo4j、开发框架等多为初识。**凡引入新工具 / 新配置 / 新框架，或完成一段非显然的工作流（环境搭建、依赖编排、调试踩坑）后，必须追加一条学习记录**——目的不是记命令，而是让所有者明白「这是什么、为什么需要、为什么这么做」。

**记录放哪（就近原则）：**
- 属于某个板块的 → 写进该板块目录下的 `DEVLOG.md`（如 `backend/DEVLOG.md`、`frontend/DEVLOG.md`）。
- 跨板块 / 基础设施 / 全局工作流（git、worktree、Docker、CI 等）→ 写进 `docs/DEVLOG.md`。
- 多窗口 / worktree 并行时，每个窗口只写自己负责板块的 DEVLOG，避免合并冲突。

**每条记录用统一模板：**
```
## YYYY-MM-DD 标题
- 做了什么：一句话
- 这是什么：涉及的工具/概念的通俗解释
- 为什么需要：不做会怎样 / 解决什么问题
- 为什么这么做：选型或做法的理由，有备选就提一句
- 踩了什么坑：报错与解决（没有可省略）
```

写法要求：用引导性、能教会人的语言，假设读者是初学者；宁可多解释「为什么」，不堆砌「怎么敲」。

## 验收硬指标（评估）

- 样本文档解析成功率 **100%**，每个 chunk 保留来源与位置。
- 实体召回率 **≥ 70%**，关系可用率 **≥ 60%**。
- 引用命中率 **≥ 70%**，明显幻觉率 **≤ 20%**。
