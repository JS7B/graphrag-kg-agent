# 前端说明

本文是 **Archigraph（档图）** 前端的**总说明**：怎么启动、代码怎么组织、设计怎么实现。
读者对象：接手维护、做评审、或想理解"为什么这么写"的人。读完应能独立改动任一部分。

> 项目名 Archigraph = archive（档案）+ graph（图谱），意为「把知识像档案归档成图谱」，
> 与招牌组件 AgentRoom 的「像素档案员」隐喻一脉相承。

---

## 1. 快速启动

    cd frontend
    cp .env.example .env     # 按需修改 VITE_API_BASE_URL（默认指向本地后端）
    npm install
    npm run dev              # 开发服务器，默认 http://localhost:5173

**联调前提**：前端只接真实后端才有意义（动画状态来自真实 RunEvent）。需要：
- 后端 `uvicorn` 起着（默认 :8000）
- Neo4j 起着（Docker Compose，共享库）
- `.env` 配好真实 LLM key

**开发预览**：`npm run dev` 后访问 `/?preview` 进 StyleGallery，用真实组件并排渲染 AgentRoom
全部 12 状态，一眼看全。`App.tsx` 里 `showGallery` 仅在 `import.meta.env.DEV` + 带 `?preview` 时挂载，不影响生产路由。

## 2. 脚本

| 命令 | 用途 |
|---|---|
| `npm run dev` | 开发服务器（热更新） |
| `npm run build` | 生产构建（`tsc -b && vite build`） |

## 3. 它是什么

**Archigraph（档图）**——一个知识图谱 GraphRAG 工作台的前端：把"文档入库 → 实体关系抽取 →
图谱构建 → 向量+图谱检索 → 带引用回答"这条链路，用三个视图可视化出来，并配一个**像素小人
AgentRoom**实时反映后端在做什么。

- 技术栈：React 19 + Vite 8 + TypeScript，图谱可视化用 Cytoscape.js
- 定位：**工作台是主体，AgentRoom 是配角**——动画只为反映程序正在执行什么，让前端丰富
- 风格：冷调 slate 中性 + 靛紫强调（对标 Linear/Notion 的工程感，刻意避开暖灰）

## 5. 设计系统（tokens.css）

所有颜色、间距、字阶、阴影都抽成 CSS 变量，集中在一处。这是"精致度"的根基——
组件里不写魔法数字，全用 token，保证全局一致。

- **表面层级**：`--color-bg`(页面底) → `--color-surface`(卡片) → `--color-surface-sunken`(下沉槽)
- **文字四档**：`--color-text` → `-body` → `-muted` → `-subtle`，从强到弱
- **强调色靛紫**：`--color-accent` 一族（hover/active/soft/border），给足交互层次
- **AgentRoom 房间色**：单独一套 `--room-*`（深紫房间）+ `--dude-*`（多彩小人）
- **签名字体**：等宽字体 `--font-mono` 承载"可溯源数据"（chunk ID、计数、时间戳），
  呼应引用可追溯护城河

## 6. 数据流（单向，红线）

```
后端 SSE ──RunEvent──> useRunEvents ──currentStage──> AgentRoom
                    └──events[]──> RunEventTimeline
```

**红线**：AgentRoom 的 `stage` **只能来自真实 RunEvent**，前端绝不伪造状态。
`useRunEvents(runId)` 订阅 SSE，累积事件并派生 `currentStage`；没后端时永远 `idle`。
这条单向流是整个动画可信度的基础，加任何状态都不能绕过它。

后端契约锁定：`Stage` 12 枚举 + 终态事件携带 `answer`。后端注释明确"改名会破坏前端状态机"，
前端这边 `types/runEvent.ts` 与之同源。**升级 Agentic RAG 后契约未变**，前端无需改动。

## 7. 三个视图

### 问答工作台（WorkbenchView，主视图）
左右两列布局：左列是 `ChatThread`(对话) + `CitationPanel`(引用证据抽屉) + `ChatComposer`(输入)；
右列是 `AgentRoom`(状态可视化) + `RunEventTimeline`(运行轨迹)。
发起问答 → `POST /api/chat` 拿 `runId` → 订阅 SSE → 终态落答案消息、引用展示最近一条。

### 文档库（LibraryView）
`GET /api/documents` 列表；上传走原生 `fetch + FormData`（绕开 apiFetch 的 JSON content-type）
→ 起 Run → SSE 订阅 → 终态刷新列表。删除走异步 Run。

### 图谱探索（GraphView）
接 `GET /api/graph/*`。**前后端字段名不一致**（后端 `name/type`，前端 `label/entityType`），
在 `api/graph.ts` 收口一层映射，不改后端。Cytoscape.js 渲染，支持实体邻域展开与模糊搜索。

## 8. AgentRoom（像素小人房间）—— 本前端招牌

### 8.1 它解决什么 / 定位
让"后端正在执行什么"变得肉眼可见。小人按状态横向飘到对应家具前，家具自己运转表达动作。
**不是装饰**，是程序执行状态的实时可视化（尤其 Agentic RAG 多轮检索时，小人在工位间
反复穿梭能直观体现 Agent 的自主决策）。

### 8.2 演进过程（为什么是现在这样）

经历了三轮迭代，每轮都是"发现上一版的根本问题、对症重构"：

**第一版 PixelAgent（已废弃）**：精细分层小人（头/身/手/道具）+ devControls 切换按钮。
问题：CSS 画精细角色一致性差、圆角化糊；且"小人当主体但没投入主体该有的逐帧动作可辨识度"。
彻底放弃。

**第二版 AgentRoom 静态小人 + 大道具**：深紫房间 + 悬浮小人 + 道具叙事。
问题：① 小人全 12 状态本体完全相同，只靠 emoji 气泡区分（AI 味）；② 单色蓝靛紫沉闷；
③ **大道具盖住小人**（碎纸机/档案柜压在身上）。属"道具挡主体"的结构性毛病。

**第三版（当前）场景叙事 + rAF 插值**：小人按状态**横向飘到对应家具工位**，家具**自己运转**
表达动作。小人本体只悬浮，全程不渲染手臂。这版同时解决了上一版的全部问题，并补上了
"中断即转"的工程要求。

### 8.3 当前机制（第三版）

**横向工位带**：房间里有 5 个固定家具 = 5 个工位：
```
电脑桌+显示器 | 咖啡角 | 打印机 | 档案柜 | 销毁台
```
12 状态归类映射到工位（如 searching/indexing → 档案柜，writing/checking → 电脑桌）。

**动作由家具运转表达**（小人本体不动）：
- 打印机吐纸（parsing/extracting）、档案柜开抽屉（indexing）、碎纸机吞纸（deleting）
- 咖啡杯持续冒热气（idle 氛围），打印机指示灯闪烁
- 家具 DOM 常驻，CSS 按 `[data-stage]` 显隐/驱动——切换不重建、不跳变

**多彩配色**：橙卫衣(`--dude-body`) + 粉高光(`--dude-body-hi`)，让小人在深紫房间跳成焦点，
不再沉闷也不与主区靛紫撞色。

### 8.4 文件分工

| 文件 | 职责 |
|---|---|
| `drawDude.ts` | 像素小人绘制：8×8 图案数组 + 编译器 → box-shadow 字符串。改图案/配色只动这里 |
| `useAgentPosition.ts` | **位置驱动**：rAF 逐帧插值 + 中断即转（见 §8.5） |
| `sceneMap.ts` | stage → {label, detail, busy} 配置 |
| `AgentRoom.tsx` | 房间结构：家具带 + 道具层 + 小人 + 状态栏 |
| `AgentRoom.module.css` | 房间/家具/小人静态样式/悬浮动画 |
| `roomScenes.css` | **道具运转**（按 `[data-stage]`）+ error 抖动。注意：位置不再由 CSS 控制 |

### 8.5 中断即转（最关键的工程决策）

**问题**：stage 快速连续变化时（Agentic RAG 一次问答会反复 searching→linking→checking），
纯 CSS `transition` **无法读取动画中间帧的真实位置**作为新起点，会导致小人瞬移/跳变。
CSS keyframes 更糟——它和 transition 互斥，切换时必跳。

**解决**：`useAgentPosition.ts` 用 JS + `requestAnimationFrame` 手动逐帧插值：
1. `currentXRef` 每帧记录小人**真实 left**（含插值中间帧），用 ref 不触发 re-render
2. `stage` 变化 → useEffect → 以 `currentXRef`（真实当前位置）为起点，rAF 逐帧到目标
3. 中途 stage 再变 → effect cleanup 取消旧 rAF → 新插值**从真实当前位置出发** = 中断即转
4. 中断时**提速到 0.4s**（正常 0.8s），减少"走回头路"的不自然感
5. 直接写 `style.left`（DOM），不进 React state——60fps 高频进 state 会拖性能

**为什么 idle 不再循环喝咖啡**：idle 曾用 keyframes 在电脑前↔咖啡杯间循环飘，这和"中断即转"
冲突最严重（keyframes/transition 互斥）。取舍：idle 固定在电脑前（left:16%），咖啡杯热气作为
**独立 CSS 氛围动效**保留（不挪动小人本体）。代价是 idle 较安静，换来中断衔接的绝对可靠。

### 8.6 box-shadow 像素法（小人怎么画的）

不用图片、不用 SVG，用 1 个 `<div>` + 多层 `box-shadow` 画整个小人：
- `drawDude.ts` 里写一个 8×8 的**字符串图案数组**（`.` 透明，其余字符映射颜色）
- `compile()` 把图案编译成一串 `"{x}px {y}px 0 0 {color}"` 投影坐标
- 渲染时 `style={{ boxShadow: DUDE_SHADOW }}`，硬边像素、性能好、易维护

这是"真栅格"做法（符合像素动画要求），改小人造型/配色**只改 PATTERN/COLOR 两个常量**。

## 9. 无障碍与性能

经 PR 审计整改（F1-F7 等），无障碍达到可公开展示/写进简历的硬指标：

- `prefers-reduced-motion`：**双层兜底**——global.css 全局关闭 CSS 动画，
  `useAgentPosition` 的 rAF 层单独检测，开启时小人直接瞬移到目标位不跑插值（F1）
- `:focus-visible`：键盘聚焦显示焦点环，鼠标点击不显示
- 语义化：`aria-label`/`aria-current` 与真实指向一致；WorkbenchView 补 sr-only `<h1>`
  （global.css 提供 `.sr-only` utility）
- 键盘可达：GraphView 的 Cytoscape canvas 不可 Tab，在实体详情空状态补**可 Tab 的实体
  按钮列表**作为并行可达路径（F5）
- 焦点管理：设置面板打开移焦到关闭按钮，关闭还焦到触发按钮（F6）
- 触控目标：引用角标按钮 ≥28px（F3）
- 颜色对比度：成功色/subtle 色加深，正文与背景满足 WCAG AA（F4）
- TopBar 状态灯接 `/health/deps` 真实连通状态，绿/红/灰三态 + aria-label（F10）
- 性能：动画优先 `transform`/`opacity`（GPU 合成）；位置插值走 rAF 直接写 DOM 不进 state；
  AgentRoom 屏幕发光仅 `data-busy=1` 时跑（省 CPU）
- Cytoscape 体积大（~650KB），是 build 警告的来源，非阻塞

## 10. 改动指南

### 加一个新状态（假设后端新增 stage）
1. `types/runEvent.ts` 的 `Stage` 加枚举值（前后端同源，后端也要加）
2. `sceneMap.ts` 加 `{label, detail, busy}` 配置
3. `useAgentPosition.ts` 的 `STAGE_X` 加目标 left 坐标
4. `AgentRoom.tsx` 若需新道具，加一个 `<div class="prop p-xxx">`
5. `roomScenes.css` 加 `[data-stage="新值"] .p-xxx { ... }` 道具动画
6. typecheck + build 验证；`?preview` 进 StyleGallery 看全 12 状态

### 改小人造型/配色
只动 `drawDude.ts` 的 `PATTERN`（图案）和 `COLOR`（颜色映射）。颜色建议改 `tokens.css`
的 `--dude-*` 变量，drawDude 里引用变量名。
