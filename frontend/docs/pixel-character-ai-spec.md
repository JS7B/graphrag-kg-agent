# 像素档案员 · AI 角色资源生成规格（最小验证版）

> 用途：指导用 AI 生成像素角色图片资源。本轮只做**最小验证**——基础站姿 + idle 呼吸帧，
> 确认 AI 能稳定产出「一致、协调、好看」的角色后，再扩展到 12 状态全套。
> 当前 CSS div 拼块方案观感不达标（圆角化、比例失调、细节糊），改走「AI 出像素图 → 前端逐帧播放」路线。

## 0. 先理解：要的不是「一堆图」，是「一致的帧」

动画 = 同一位置快速切换帧。**所有帧必须：同尺寸、角色同站位、同造型、同配色、同光源**。
任何一帧角色大小/位置/风格变了，播放就会抖动变形。**一致性是第一命脉**，AI 生成最难的也是这点。

本轮只要 3 张图即可验证：
1. 角色基础站姿（idle 第 1 帧 / 静止）
2. idle 呼吸第 2 帧（躯干/肩部轻微起伏，其余不变）
3. （可选）一张正面或微侧角度备选，挑更顺眼的定为基准

## 1. 角色设定（档案管理员）

> v2 调整（基于首轮生成反馈）：① 角色明显年轻化（首轮偏老，像 35+，要做成 20 出头）；
> ② 工装改为**蓝靛紫卫衣**（hoodie，更年轻亲和，蓝紫之间，呼应产品靛紫主色不冲突）；
> ③ 眼镜加**像素化反光**（镜片左上角 1-2 个固定白色高光块示意反光，非写实渐变）。

- **身份**：知识图谱系统的档案管理员吉祥物，专业、亲和、利落。
- **造型**：戴方框眼镜的**年轻**档案员（20 出头，脸圆润、眼睛大而有神、表情开朗）；整洁短发偏分；穿**蓝靛紫连帽卫衣**。
- **体型**：适中略 Q——头略大于真人比例但不夸张，约 1 : 1.5 头身比；圆润但不臃肿。
- **朝向**：三分面（约 3/4 视角）——比纯正面生动，又比纯侧面好认。**注意：同一动画的多帧必须同一视角同一站位**，不要混用三分面和正面。
- **辨识特征**（务必稳定出现）：方框眼镜（带像素反光高光）、偏分短发、蓝靛紫卫衣。这三点是「同一个角色」的锚。
- **配色锚点**：卫衣 = 蓝靛紫（介于 `#5b6ee1` 蓝 与 `--color-accent` 靛紫之间），与 UI 主色呼应。

## 2. 技术规格

| 项 | 要求 |
|---|---|
| 风格 | 像素艺术（pixel art），细像素、多细节但**边缘硬朗**（无抗锯齿模糊） |
| 画布 | 正方形，建议 **128×128** 或 256×256（统一，所有帧一致） |
| 角色占画面 | 约 70%，**居中**，脚底留一致的空白（便于对齐地面线） |
| 背景 | **纯透明**（首选 PNG alpha）；做不到就**纯白或纯单色**，且每张完全相同 |
| 配色 | 有限色板（像素画精髓）：靛紫工装（呼应 `--color-accent`）、肤色、深色头发/镜框；**纯色平涂 + 一档暗部**，不要渐变、不要半透明 |
| 光源 | 统一左上方来光（暗部一致落在右下） |
| 输出 | 每帧单独 PNG；命名 `idle_01.png` / `idle_02.png` |

## 3. AI 提示词（可直接用）

3/4 view facing slightly right, round soft youthful face with big bright eyes,
square black-framed glasses with a small white reflection highlight on the
upper-left of each lens, neat side-parted short dark hair, wearing a
BLUE-INDIGO hoodie (between blue and indigo-purple), soft rounded proportions
(head slightly large, ~1:1.5 head-to-body), standing idle pose, full body,
centered, limited color palette, flat colors with one shade of shadow,
hard pixel edges, no anti-aliasing, top-left light source, transparent
background, 128x128, clean game sprite, high quality pixel art
```

**中文（备用）：**

其余部位不变 — same character, identical style, position and view angle,
only slight chest/shoulder rise for breathing, everything else unchanged
```

## 4. 一致性技巧（多帧关键）

- **固定随机种子（seed）**：同一 seed + 微调描述，角色更易保持一致。
- **用参考图（image-to-image / reference）**：先定好第 1 张满意的，后续帧用它当参考图生成，只描述「改什么动作」。
- **描述里锁死不变项**：每次都写全「方框眼镜 + 偏分短发 + 靛紫工装 + 同站位」，别省。
- **一次生成多张挑**：同提示出 4 张挑最一致好看的。

## 5. 验收标准（拿到图怎么判断能不能用）

- [ ] 角色**居中**、脚底留白一致（两帧叠放角色不偏移）
- [ ] 两帧角色**大小、造型、配色完全一致**，只有呼吸部位有差异
- [ ] 真·像素感：边缘硬朗、纯色、无渐变模糊
- [ ] 三个辨识特征（眼镜/偏分发/靛紫工装）清晰且一致
- [ ] 背景透明或纯色可干净抠除
- [ ] 比例协调：不头重脚轻、不臃肿（对照当前 CSS 版的「大头胖身细腿」问题，新版要明显更协调）

## 6. 验证通过后的下一步（本轮不做，仅说明）

若上面 2-3 张验证 OK：
1. 大脑据此扩写 **12 状态全套**的提示词清单（每状态 2-4 帧）。
2. 前端把帧图接入 `PixelAgent`：用 `steps()` 逐帧播放，或拼成 sprite sheet。
3. `stage` 仍由真实 `RunEvent` 驱动（红线不变，见 `pixel-agent-guide.md`）。
若 AI 始终做不出一致角色：退回「粗像素 + 正面 + CSS」方案（大格子、克制细节，CSS 能做好）。

## 7. 后续道具构想（基础角色定稿后再做，先记录方向）

12 状态动画的精髓就是**角色 + 道具**的组合。基础角色定稿后，为相应 stage 设计手持道具
（与角色同风格、同色板、同视角）：

- `searching` / `checking`：放大镜（翻找 / 校对引用）
- `parsing`：扳手 或 拆包刀（拆文件）
- `writing`：钢笔 / 键盘（打字输出答案）
- `indexing`：文件夹 / 档案盒（整理档案柜）
- `extracting` / `linking`：标签贴纸 / 连线
- `deleting`：碎纸机
- `rebuilding`：复印机

道具应作为**独立帧元素**与角色组合，便于复用。生成时强调「同一角色、同一视角、手持 XX 工具」。
