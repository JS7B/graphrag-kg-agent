# 像素 Agent 房间式改造规格

> 方向转变（2026-06-24）：放弃"精细角色 + 12 套逐帧动画"路线（AI 角色一致性难、CSS 拼精细小人观感差），
> 改用 **ZCodeRoom 式"极简悬浮小人 + 场景道具叙事"**。灵感来自 `E:\Mine\ZCodeRoom-原型.html`。
> 范围：**只改造工作台侧栏的「像素 Agent 区 + 运行事件时间线」**，三视图工作台主体不动。

## 核心理念:小人弱化，场景叙事

| 维度 | 旧方案（废弃） | 新方案 |
|---|---|---|
| 小人 | 精细五官/四肢/发型/眼镜，22+ 层 div | **极简色块人**（~8×8 网格，box-shadow/div 像素法），悬浮 |
| 小人动作 | 12 套复杂逐帧动画 | **只有上下浮动（bob）** + 工作时轻微摆动，四肢不动 |
| 状态表达 | 靠角色自己做动作 | 靠**头顶气泡图标 + 周围场景道具**（文档闪过、碎纸机出现、放大镜浮现） |
| 实现难度 | 极高（要美术资源） | **低**（纯 CSS 色块 + 简单 keyframe，原型已验证可行） |

**一句话**：小人就是个会呼吸的可爱色块，"它在干什么"全靠它**周围发生的事**来讲。

## 数据流（已就绪，不变）

```
后端 RunEvent(SSE) → useRunEvents(runId) → currentStage + events → AgentRoom
```
`WorkbenchView` 已用 `apiFetch('/api/chat')` 起 Run、`useRunEvents` 订阅真实 SSE。
房间只是 `currentStage` 的新表现层。**红线不变：stage 只来自真实 RunEvent。**

## 12 状态 → 场景道具映射

每个 stage = 小人不变（悬浮+可能轻微摆动）+ 头顶气泡图标 + 周围一个场景道具/动效：

| stage | 气泡 | 场景道具/动效 |
|---|---|---|
| idle | ☕ | 安静，小人慢悠悠浮动，偶尔眨眼 |
| uploading | 📥 | 文档从右侧"飞入"房间 |
| parsing | 📄 | 文档在小人面前快速翻页/拆开 |
| extracting | 🏷️ | 实体标签贴纸从文档上"弹出" |
| linking | 🔗 | 两个节点间画出连线 |
| indexing | 🗂️ | 档案柜抽屉开合、文件归位 |
| searching | 🔍 | 放大镜浮现，文件堆里扫描光 |
| checking | ✓ | 放大镜对着文档逐行扫 |
| writing | ⌨️ | 文字/纸条从打字机吐出 |
| deleting | 🗑️ | 碎纸机出现，文档被吸入、碎屑飞 |
| rebuilding | 🔄 | 复印机出现，文档复制重排 |
| error | ⚠️ | 红色感叹号气泡，小人抖动一下 |

**关键**：小人本体在所有状态几乎一样（省去逐帧角色动画），变的是**气泡 + 道具**。这就是降维的精髓。

## 实现要点（给实现者）

1. **新建 `AgentRoom` 组件**替换 `PixelAgent`：`src/components/AgentRoom/`。
   - `AgentRoom.tsx`：房间容器 + 小人 + 气泡 + 道具层（道具 DOM 常驻，CSS 按 `data-stage` 显隐，避免切换重建跳变——沿用原 PixelAgent 已有的好做法）。
   - `AgentRoom.module.css`：房间背景/地面/小人静态形态。
   - `roomScenes.css` 或 `animations.css`：各 stage 的道具显隐 + 动效，用 `[data-stage]` 选择器。
   - `sceneMap.ts`：stage → { label, bubbleIcon, 进度文案 }（替换现有 stageMap）。
2. **小人极简**：参考 ZCodeRoom 原型的 `drawDude()`——8×8 网格色块，配色走 design token（蓝靛紫呼应品牌）。一个 `bob` 浮动 keyframe 全状态通用。
3. **道具用纯 CSS 块**：碎纸机/放大镜/文档都是简单矩形+圆角+token 配色拼的，配 `@keyframes` 做"闪过/吸入/扫描"。参考 ZCodeRoom 原型里 monitor/desk/door 的画法。
4. **保留运行轨迹**：房间下方保留事件时间线（可缩简），两者同属一个 Agent 状态区。
5. **WorkbenchView 改动**：`sideCol` 里把 `<PixelAgent>` 换成 `<AgentRoom stage={currentStage} events={events} />`，其余不动。
6. **devControls 保留**：开发预览切 stage 看房间效果（红线：仅开发用）。

## 取舍说明

- **为什么不做整个工作台房间化**：graphrag-kg-agent 定位是专业文档理解工作台（简历核心价值），三视图需要信息密度与实用性。房间作为"Agent 状态区"嵌入，既得生动感又不牺牲专业性。
- **为什么小人不做精细动画**：精细角色一致性难（AI 路线验证受阻）、CSS 拼精细小人观感差（已验证）。极简小人 + 场景叙事用最低成本达到"看得见 Agent 在干活"的效果，且更耐看、更好维护。
- **像素美术**：极简色块人 + CSS 道具，**不需要外部美术资源**，纯前端可做。日后若要精致化，道具可逐个换成像素图。
