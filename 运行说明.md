# 运行说明

本地启动 graphrag-kg-agent 的完整步骤。三个部分：**Neo4j（Docker）→ 后端（FastAPI）→ 前端（Vite）**，按顺序起。

> 环境：Windows + Anaconda（`myself` 环境，Python 3.12）+ Docker Desktop + Node.js。

---

## 0. 前提检查（首次启动）

| 依赖           | 检查                                                              | 没有怎么办                                        |
| -------------- | ----------------------------------------------------------------- | ------------------------------------------------- |
| Docker Desktop | 托盘鲸鱼图标绿色 = Engine running                                 | 打开 Docker Desktop 等它就绪                      |
| 根目录`.env` | 已存在（含 Neo4j 密码 + LLM 配置）                                | `cp .env.example .env` 后填真实值               |
| 后端依赖       | `myself` 环境已装 fastapi/neo4j/openai/sse-starlette/PyMuPDF 等 | 见下方「后端」步骤                                |
| 前端依赖       | `frontend/node_modules` 存在                                    | **首次必须** `cd frontend && npm install` |

---

## 1. 启动 Neo4j（Docker）

在仓库根目录：

```powershell
docker compose up -d neo4j
```

- 验证：浏览器开 http://localhost:7474 ，用 `neo4j` / `<.env 里的 NEO4J_PASSWORD>` 登录。
- 端口：7474（Browser）、7687（Bolt）。
- 数据存在 `./neo4j/data`（已 gitignore）。停止：`docker compose stop neo4j`（数据保留）。

---

## 2. 启动后端（FastAPI）

新开一个终端，进 `backend` 目录，激活 `myself` 环境：

```powershell
conda activate myself
cd backend
python -m uvicorn app.main:app --reload --port 8000
```

- API 地址 http://localhost:8000 ；交互文档 http://localhost:8000/docs 。
- **验证健康**：
  - http://localhost:8000/health → `{"status":"ok"}`
  - http://localhost:8000/health/deps → `{"neo4j":"ok","llm":"configured"}`（Neo4j 起着 + LLM 配了真 key 时）
- 启动时会自动建图谱约束 + 向量索引（Neo4j 没起也只告警不崩）。
- **鉴权**：若 `.env` 配了 `API_KEY`，非 `/health` 接口需带请求头 `X-API-Key: <你的key>`；为空（默认）则不鉴权。

> 若 `myself` 环境缺依赖：`pip install -r requirements.txt`

---

## 3. 启动前端（Vite）

再开一个终端，进 `frontend` 目录：

```powershell
cd frontend
npm install        # 首次必须；之后可跳过
npm run dev
```

- 前端地址 http://localhost:5173 。
- 前端默认连后端 `http://localhost:8000`（可在 `frontend/.env` 用 `VITE_API_BASE_URL` 改，一般不用）。

---

## 4. 端到端试用

1. 浏览器开 http://localhost:5173 。
2. **文档库**：上传一个 `.md` / `.txt` / `.pdf`（可用 `samples/` 里的 `eval-agents.md`）。看像素档案员房间跟着 uploading→parsing→extracting→indexing 走。
3. **问答**：在工作台提问（如「项目用什么图谱数据库？」），看房间 searching→checking→writing，答案带引用角标，点角标看来源 chunk。
4. **图谱**：图谱视图查看抽取出的实体与关系。
5. **设置**：设置页看 Neo4j / LLM 连通状态。

---

## 5. 评估（可选）

```powershell
cd backend
D:\Anaconda3\envs\myself\python.exe ..\evals\run_eval.py
```

产出 `evals/report.md`（解析成功率 / 实体召回 / 关系可用 / 引用命中 / 幻觉率）。需 Neo4j 起着 + LLM 配好。详见 `docs/evaluation.md`。

---

## 常见问题

- **`/health/deps` 的 llm 显示 `not_configured`**：`.env` 里 OPENAI_* 还是占位值，填真实的 base_url / api_key / 模型名。问答/抽取需要真实 LLM。
- **rerank 报错**：不影响。RERANK_MODEL 不可用时问答会自动降级为按向量分数排序，主链路照常。
- **前端图谱/问答空白**：先确认后端在跑（8000）、Neo4j 在跑（7687）、且已上传过文档（图库有数据才有图谱）。
- **端口被占**：后端 8000 / 前端 5173 / Neo4j 7474·7687，确认没有别的进程占用。
- **CHAT_MODEL 选型**：实体关系抽取用 JSON 模式（`response_format json_object`），需选支持该模式的模型。
